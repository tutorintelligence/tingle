# Coded-symbol chirp protocol (v2 signaling)

Status: IN PROGRESS (branch feat-coded-symbols). Hard cutover approved by
Josh 2026-07-11 — no dual decoding, no backward compat; one Flash EP
switches the world. The Goertzel tone decoder is deleted in the PR that
ships this (after live verification per the CLAUDE.md iteration loop).

## Why

The v1 protocol (pure 80ms tones at 17.5/18/18.5/19kHz, threshold-crossing
detection) needed ~10 interacting heuristics to survive real conditions
(see GoertzelDetector.swift history, 2026-07-11). Root cause: per-window
narrowband energy is a weak detection statistic. Matched filtering of
coded waveforms buys ~19dB of processing gain (time-bandwidth product
0.08s x 1kHz = 80) and makes the failure classes structural non-events:
noise pops, lo-fi intermod artifacts, speech, and burst fragmentation do
not correlate with a chirp template.

## Symbols

4 sample slots on the device = 4 symbols. Linear chirps, 2 bands x 2
directions (up/down in the same band are near-orthogonal at TB=80;
cross-band is fully disjoint):

  S0  16.75 -> 17.75 kHz  up      slot 0
  S1  17.75 -> 16.75 kHz  down    slot 1
  S2  18.25 -> 19.25 kHz  up      slot 2
  S3  19.25 -> 18.25 kHz  down    slot 3

80ms, 5ms raised-cosine edges, 0.95 amplitude, 48kHz mono 16-bit WAV.
Generated by Flasher (replaces the pure-tone generator); the device
payload is UNCHANGED (it just triggers slots — content is ours).

## Pair coding (unchanged semantics, trivial now)

The device sequences the same pairs as v1 (single = white press,
(N,N+1) = mode, (N,N-1) = fx, (0,2)/(2,0) = trigger down/up,
(1,3)/(3,1) = beacon released/held). With ~19dB symbol margins, pairing
is a ten-line window match, not a defensive state machine.

## Decoder (SymbolDetector.swift)

- Streaming normalized cross-correlation against the 4 templates
  (vDSP FFT overlap-save; block 8192, hop 4096 at 48k).
- Detection: normalized peak >= threshold AND dominance over the
  best other template; correlation magnitude feeds the same level
  self-calibration as v1.
- KEPT from v1 (transport-agnostic): pilot acquisition lock (3 periodic
  consistent beacons; staleness unlock; fast re-lock), absolute
  plausibility floor, beaconSensed presence-without-state, level
  reference gates.
- DELETED with v1: hysteresis, peak-bound sustain, moving-tone veto,
  guard-bin machinery, overlap early-close, post-beacon lone gates.

## Test plan (harness is the referee)

- Synthetic: symbol decode floors (expect >=15dB better than v1 pins),
  pair decoding, acquisition lifecycle, 90s noise soak.
- Adversaries: v1 corpus PLUS pure tones at the old frequencies (legacy
  tones must NOT decode — they don't match templates).
- Fixtures: v1 tone fixtures retire with the Goertzel decoder. New
  real-recording fixtures captured after the first live flash (audio
  review rule: Josh approves every committed recording).
- Live: dev.sh -> Josh verdict -> merge (per iteration loop).
